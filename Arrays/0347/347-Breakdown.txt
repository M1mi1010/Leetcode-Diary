Final attmept:
- Used a priority queue with a comparator that checks two arrays index 1 values. The priority queue stores integer arrays, where arr[0] is the number and arr[1] is the frequency
- Then using the frequency maps entry set, add the entries key and value as a new array to the priority queue
- Then if the size of the heap after adding is more than k, remove the item
- Then pop everything off the queue, writing each arrays index 0 to results until k elements are found

Why cant I break when heap is size k?
- heap.size() < k → accept candidates
- heap.size() == k → still keep checking, because a better candidate may arrive
- heap.size() > k → remove the weakest candidate
